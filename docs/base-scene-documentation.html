BaseScene Documentation

Overview

BaseScene is the foundation for all game scenes in our shooter game. It handles core functionality like physics setup, music, player creation, and game state management.

Scene Initialization Flow

The create() method has been refactored into smaller, focused methods for better organization and maintainability.

1. Basic Setup

#disableRightClickMenu()

Disables browser context menu for better game experience

#setupCanvasDimensions()

Gets game canvas dimensions for proper scaling

#initializeManagers()

Sets up all game managers (sound, entities, etc.)

2. Audio & Input Setup

#setupMusic()

Initializes background music with proper volume settings

#enableKeyboardInput()

Enables keyboard controls for player interaction

3. World Setup

#setupWorldPhysics()

Configures game physics boundaries and settings

#createParallaxBackground()

Creates scrolling background layers

#createPlatforms()

Sets up game platforms for player interaction

#storeSpawnInfo()

Configures spawn points for players and entities

Private vs Public Methods

BaseScene uses a mix of private and public methods for better code organization:

Private Methods (Internal Use)

// Only BaseScene can use this
#setupMusic() {
    const bgMusic = this.sound.add('bgMusic');
}

Public Methods (External Use)

// Any scene can use this
getSpawnHeight() {
    return this.groundTop - 16;
}

Game State Management

this.sceneState = {
    isReady: false,
    isPlaying: false,
    isLoading: true,
    startTime: Date.now(),
    levelData: {
        currentScore: 0,
        enemiesDefeated: 0,
        timeElapsed: 0,
        checkpoints: []
    }
}

Example Usage

// In a custom game scene:
class MyGameScene extends BaseScene {
    create() {
        // Call parent create method first
        super.create();
        
        // Add custom scene setup
        const spawnY = this.getSpawnHeight();
    }
}

Best Practices

  • Always call super.create() first in child scenes
  • Use private methods (#) for internal scene logic
  • Use public methods for functionality needed by other scenes
  • Keep methods focused on single responsibilities
  • Use the scene state for tracking game progress
  • Handle errors appropriately in each method
