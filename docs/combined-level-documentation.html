CombinedGtuLevel Documentation

Overview

The CombinedGtuLevel is a sophisticated level management system that extends BaseScene to create a dynamic, progressively-loading game environment. It handles multiple concatenated levels, enemy management, physics, and various game mechanics.

Key Features

Progressive Loading

  • Sections load dynamically as player progresses
  • Optimized memory usage
  • Smooth level transitions

Level Management

  • Multiple level sections
  • Checkpoint system
  • Dynamic world bounds

Entity Management

  • Enemy tracking and spawning
  • Bullet physics and pooling
  • Interactive objects

Technical Implementation

Initialization

constructor() {
    super({ key: 'CombinedGtuLevel' });
    // Level tracking
    this.currentLevel = 0;
    this.totalEnemies = 21;
    
    // Progressive loading properties
    this.loadedSections = new Set();
    this.sectionWidth = 640;
    this.lastLoadedSection = -1;
}

Level Loading System

Property Description Default Value
sectionWidth Width of each loadable section 640 pixels (20 tiles)
bufferDistance Distance before level end to start buffering next section 200 pixels
singleLevelWidth Width of a complete level section 2048 pixels
Note: The level system uses LDtk (Level Designer Toolkit) for level design and management.

Asset Management

Required Assets

  • Tilesets: GtuTileset.png
  • Character sprites: Idle, Jump, Crouch, Death, etc.
  • Enemy sprites: Warriors, Slimes, Drones
  • Objects: Trampolines, Bitcoins, Bullets
  • Audio: Background music, sound effects

Physics and Collision System

// Physics groups initialization
this.platforms = this.physics.add.staticGroup();
this.enemies = this.physics.add.group();
this.bullets = this.physics.add.group({
    classType: Bullet,
    maxSize: 20,
    runChildUpdate: true,
    allowGravity: false
});

Collision Layers

  • Ground Layer: Main collision layer for solid surfaces
  • Platform Layer: Separate layer for platforms and special tiles
  • Entity Collisions: Managed through physics groups

Enemy Management

Important: Enemy spawning is tied to level sections and managed by the EnemyManager class.

Enemy Types

  • Melee Warriors: Close-range combat enemies
  • Slimes: Basic movement patterns
  • Drones: Flying enemies with projectile attacks
  • Turrets: Stationary enemies with ranged attacks

Performance Optimization

Key Strategies

  • Progressive section loading to minimize memory usage
  • Object pooling for bullets and particles
  • Efficient tile rendering through layer management
  • Entity cleanup for out-of-view sections

Integration with Game Systems

Manager Classes

  • CollisionManager: Handles all physics interactions
  • TextStyleManager: Manages text styling and UI
  • EnemyManager: Controls enemy spawning and behavior
  • EffectsManager: Handles particles and visual effects
  • MusicManager: Controls game audio and sound effects

Event System

// Event handling example
eventBus.on('enemyDefeated', () => {
    this.remainingEnemies--;

        this.triggerVictory();
