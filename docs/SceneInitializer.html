SceneInitializer Documentation

Overview

The SceneInitializer class is responsible for setting up and initializing various components of a game scene in the ShooterGTU game. It handles essential setup tasks such as camera configuration, physics setup, game object group creation, player initialization, and manager instantiation.

Constructor

constructor(scene)

Parameters:
  • scene: The Phaser scene instance to be initialized

Methods

initializeScene()

Main initialization method that orchestrates the entire scene setup process. Calls all other setup methods in sequence.

Returns: An object containing all initialized managers

setupCamera()

Configures the main camera for the scene with the following settings:

  • Sets viewport to 1920x1080
  • Configures camera bounds based on room dimensions
  • Sets background color to black
  • Calculates and applies appropriate zoom level
  • Centers the camera on the room

setupPhysics()

Initializes the physics system:

  • Enables collision bounds on all sides
  • Sets physics world bounds according to room dimensions

createGroups()

Creates and initializes various physics and non-physics groups for game objects:

  • enemies: Physics group for enemy entities
  • slimes: Physics group for slime entities
  • drones: Physics group for drone entities
  • bitcoins: Regular group for collectibles
  • bullets: Physics group with Bullet class type (max 10 bullets)
  • platforms: Static physics group for platforms

createPlayer()

Creates and initializes the player character:

  • Spawns player at 10% of screen width on X-axis and Y position 100
  • Sets initial velocity to zero
  • Stores spawn point for debug purposes

setupDebug()

Initializes debugging tools:

  • Creates debug graphics
  • Instantiates the debug system

setupManagers()

Creates and returns an object containing all necessary game managers:

  • stateManager: Handles game state
  • effectsManager: Manages game effects
  • enemyManager: Controls enemy behavior and spawning
  • collisionManager: Handles collision detection and response
  • boundaryManager: Manages scene boundaries
  • animationManager: Controls animations

Dependencies

The SceneInitializer relies on several other classes and managers:

  • Player
  • Bullet
  • StateManager
  • EffectsManager
  • EnemyManager
  • CollisionManager
  • SceneBoundaryManager
  • AnimationManager
  • DebugSystem

Usage Example

class GameScene extends Phaser.Scene {
    create() {
        const sceneInitializer = new SceneInitializer(this);
        const managers = sceneInitializer.initializeScene();
        
        // Access managers as needed
        this.stateManager = managers.stateManager;
        this.enemyManager = managers.enemyManager;
        // etc...
    }
}
