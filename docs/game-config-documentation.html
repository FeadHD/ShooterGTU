Game Configuration Documentation

Overview

This documentation details the core configuration settings for the game engine. The configuration defines the fundamental behavior, rendering options, and scene management for the Phaser 3 game instance.

Configuration is defined in src/config/config.js and is used to initialize the main Phaser.Game instance.

Renderer Configuration

export const config = {
    type: Phaser.AUTO,
    pixelArt: true
};
Property Value Description
type Phaser.AUTO Automatically chooses between WebGL and Canvas rendering based on browser support. WebGL is preferred for better performance.
pixelArt true Enables crisp pixel art rendering by disabling texture smoothing and using nearest-neighbor scaling.
API Reference: See Phaser.Core.Config

Scale Configuration

scale: {
    mode: Phaser.Scale.NONE,
    parent: 'game-container',
    width: 1920,
    height: 1080,
    autoRound: true,
    zoom: 1,
    autoCenter: Phaser.Scale.CENTER_BOTH,
    fullscreenTarget: 'game-container',
    expandParent: true
}
Property Value Description
mode Phaser.Scale.NONE No scaling applied. The game canvas maintains its exact dimensions.
parent 'game-container' ID of the HTML element that will contain the game canvas.
width 1920 Base width of the game canvas in pixels (1080p resolution).
height 1080 Base height of the game canvas in pixels (1080p resolution).
autoRound true Automatically rounds scaled dimensions to prevent sub-pixel rendering.
zoom 1 Initial zoom level of the game canvas.
autoCenter Phaser.Scale.CENTER_BOTH Centers the game canvas both horizontally and vertically within its parent container.
fullscreenTarget 'game-container' Element to be used for fullscreen mode.
expandParent true Automatically expands the parent container to fit the game canvas.
API Reference: See Phaser.Scale.ScaleManager

DOM Configuration

dom: {
    createContainer: true
}
Property Value Description
createContainer true Creates a DOM container for HTML elements. Required for UI overlays and HTML-based interfaces.
API Reference: See Phaser.DOM

Physics Configuration

physics: {
    default: 'arcade',
    arcade: {
        debug: false,
        gravity: { y: 400 },
        tileBias: 32
    }
}
Property Value Description
default 'arcade' Uses Arcade Physics system, optimized for AABB (Axis-Aligned Bounding Box) collision detection.
debug false When true, shows physics bodies and velocity vectors for debugging.
gravity.y 400 Vertical gravity force in pixels per second squared. Positive values pull downward.
tileBias 32 Extra padding for tile collision detection, helps prevent sprite tunneling.
API Reference: See Phaser.Physics.Arcade

Input Configuration

input: {
    mouse: {
        preventDefaultWheel: true,
        preventDefaultContextMenu: true
    }
}
Property Value Description
preventDefaultWheel true Prevents the browser's default wheel behavior, useful for custom scroll implementations.
preventDefaultContextMenu true Disables the browser's context menu on right-click within the game canvas.
API Reference: See Phaser.Input.InputManager

Scene Configuration

The game includes multiple scenes organized by type:

Core Scenes

  • Preloader - Asset loading and initialization
  • TitleScene - Main title screen
  • IntroScene - Game introduction

Menu Scenes

  • MainMenu - Primary navigation
  • Settings - Game settings
  • ControlsSettings - Input configuration
  • SoundSettings - Audio settings
  • PauseMenu - In-game pause
  • Leaderboard - High scores
  • Credits - Game credits

Game Scenes

  • GameScene1-5 - Main game levels
  • Matrix640x360 - Special matrix level
  • BlueTest - Test environment
  • GtuTestLevel0-2 - GTU test levels
  • CombinedGtuLevel - Combined test level

Development Scenes

  • DevHub - Developer tools and testing
  • SoundTester - Audio testing utility
Scenes are loaded in the order they appear in the configuration. The Preloader scene is loaded first to ensure all assets are available before other scenes initialize.
API Reference: See Phaser.Scenes.SceneManager

Best Practices

Important Considerations:
  • Keep debug mode disabled in production for better performance
  • Adjust gravity and tileBias based on game physics requirements
  • Consider device capabilities when setting resolution and scale mode
  • Organize scenes in a logical order for proper loading sequence
