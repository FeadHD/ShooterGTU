Boot Scene Documentation

Overview

The Boot scene is the initial scene in ShooterGTU's game lifecycle. It serves as the entry point for the game, handling the basic setup required before any assets are loaded. This scene is responsible for configuring the base URL for asset loading and transitioning to the Preloader scene.

Core Functionality

export class Boot extends Scene {
    constructor() {
        super('Boot');
    }

    preload() {
        this.load.setBaseURL(window.location.href);
    }

    create() {
        this.scene.start('Preloader');
    }
}
Method Description
constructor() Initializes the Boot scene with the key 'Boot'
preload() Sets the base URL for asset loading to the current window location
create() Transitions to the Preloader scene once initialization is complete

Scene Flow

Stage Description
1. Initialization Scene is created and registered with Phaser's scene manager
2. URL Configuration Base URL is set for asset loading during preload phase
3. Transition Scene transitions to Preloader scene for asset loading

Usage Notes

  • The Boot scene should be the first scene added to the game configuration
  • No game assets should be loaded in this scene - asset loading is handled by the Preloader
  • The scene's primary purpose is to set up the asset loading URL and quickly transition to the Preloader
  • Minimal processing should occur in this scene to ensure fast game startup

Best Practices

  • Keep the Boot scene lightweight and focused on initialization
  • Avoid adding game logic or asset loading in this scene
  • Ensure proper error handling for URL configuration
  • Maintain the simple transition flow to the Preloader scene
